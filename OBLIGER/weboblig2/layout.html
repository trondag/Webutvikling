Layout

Det er mange forskjellige måter å lage layout til et HTML dokument. Jeg går gjennom noen av disse, men tar først for meg gamle metoder.

CSS level 1

Selve starten til CSS begynner i det sekundet Håkon Wium Lie sender en epost til WC3 i 1994. Tiden etter er historien om irriterende floats, svetting foran dataen og en hensynsløs og blodig krig mellom nettlesere.

Jeg vil her ikke gå gjennom "the dark ages", det kapittelet blir litt for mørkt for meg. Jeg begynner heller i CSS level 1

Før brukte man ikke egne stilark og slikt, man koda CSS inline i HTML koden. Width og height, float og clear, margin og padding kom på denne tiden (90-tallet). Disse attributtene var banebrytende på denne tiden, rett og slett en revolusjon innen web-design. Nå trengte man ikke lenger å bruke br-tagger overalt for å bryte linje.

CSS level 2

I 1998 ble level 2 lansert, og vi fikk table-taggen. Man startet med å benytte egne stilark for CSS, CSS ble nemlig definert på denne tiden. Derfor kunne man begynne å skille design og innhold fra hverandre. Desverre for table-taggen, tok det tid før den ble støtta i IE. Webutviklere hacka nettsidene sine med float, clearfix og position: absolute. Mens alle utviklere svetta og bantes over floatingen sin, satt en gruppe utviklere i WC3 og jobbet på noe stort.

Litt at problemet på denne tiden, var at teamet bak CSS inmplementerte en rekke ting, men browsere ville ikke ha egenskapene i nettleserne. Det var derfor en rekke bugs og feil.
