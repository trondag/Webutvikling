GIT

I 2005 lagde Linus Torvalds et dataprogram. Programmet er designet for versjonskontroll, og det å spore hvilke endringer som blir gjort på en fil.

GIT er også et kjekt verktøy for utviklere, da man kan ha en kode som alltid fungerer som main-branch, og lage nye branches som flere kan jobbe/eksperimentere på.

En branch kan senere brukes til å implementere ny kode inn i et program.

Her kan bedriften også ha full kontroll med hvilke endringer som blir gjort, hvem som har gjort det, og de kan også bestemme hvilken kode de vil benytte.

Github

Github er et web-basert program som bruker GIT til versjonskontroll. Github er en enkel måte å organisere for eksempel et nettsted som flere utviklere jobber med.

Jeg har nå selv prøvd å bruke Github, og jeg har lasta opp nettsiden min dit (commit). Hvis jeg laster opp en ny versjon, vil det stå informasjon om det

Jeg synes det var enkelt å laste opp filer på github, jeg ser fordelen av å ha kontroll over tidligere versjoner og hvilke endringer som er blitt gjort. Da kan man også lettere finne feil i koden.
